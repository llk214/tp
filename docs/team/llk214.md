# Liu Lekuan - Project Portfolio Page

## Overview

RLAD is a minimalist **Command-Line Interface (CLI) finance tracker** built for power users who prefer the speed and
efficiency of a terminal. It allows users to track income/expenses, set budget goals, and get instant summaries with
simple, intuitive commands.
My primary contributions were the **filter, sort, and storage management** features, which together give users
powerful tools to search, organize, and export their financial data.

### Enhancements implemented

- **FilterCommand:** Standalone `filter` command with a shared `buildPredicate()` method reused by `list`, `delete`,
  and `summarize`. Supports `--type`, `--category`, `--amount` (with operators `-gt/-gte/-eq/-lt/-leq`), `--date`,
  `--date-from`, `--date-to`. Uses predicate chaining (`Predicate.and()`) so filter logic is composable and maintained
  in a single source of truth across all commands.

- **Enhanced FilterCommand** (PR #53): Added partial and case-insensitive category matching, multi-category filtering
  (comma-separated), category codes (1–12), relative date keywords (`today`, `yesterday`, `this-week`, `this-month`,
  `last-month`, `last-year`), and date-range validation (`from <= to`). Relative dates required `LocalDate`
  calculations (e.g., `this-week` uses `with(DayOfWeek.MONDAY)`). 45 unit tests cover all filter paths.

- **SortCommand & TransactionSorter:** Global sort state (`sort amount desc`) persists across commands. Per-command
  `--sort` flag on `list`/`filter` overrides global sort for that invocation only. Supports `amount`/`date` fields,
  `asc`/`desc` directions, `reset`, and `show`. `TransactionSorter` is a stateless utility; global state lives in
  `TransactionManager`, so one-off sorts do not mutate the global order.

- **Storage Management** (PR #57): `export`, `import`, and `clear` commands with `CsvStorageManager`. Export writes
  all transactions to CSV; import supports merge or replace mode; clear prompts for confirmation (or `--force`). CSV
  parsing handles quoted fields, escaped quotes, commas in descriptions, and newlines. Import validates each row
  independently — malformed rows are skipped with error messages while valid rows are imported.

- **Assertions & Logging** (PR #39): Added `assert` statements and `java.util.logging` to `SortCommand` and
  `TransactionSorter` for defensive programming.
