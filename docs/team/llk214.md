# Liu Lekuan - Project Portfolio Page

## Overview

RLAD is a minimalist **Command-Line Interface (CLI) finance tracker** built for power users who prefer the speed and
efficiency of a terminal. It allows users to track income/expenses, set budget goals, and get instant summaries with
simple, intuitive commands.
My primary contributions span the **filter, sort, storage management, autosave, help, and error handling** features,
together with extensive bug fixing and documentation work across the project lifecycle.

### Enhancements implemented

- **FilterCommand** (PR [#53](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/53)): Implemented
  `FilterCommand` as a shared filtering utility consumed by `list` and `summarize`. The user-facing interface
  uses colon syntax so filters compose naturally on a single line:

  | Filter | Example | Effect |
  | :--- | :--- | :--- |
  | `type:` | `list type:debit` | show only debits or credits |
  | `cat:` | `list cat:food` | case-insensitive substring match on category |
  | `from:` / `to:` | `list from:2026-03-01 to:2026-03-31` | inclusive date range |
  | `min:` / `max:` | `list min:10 max:200` | amount range |

  Input is normalised before parsing so `cat: food` (space after colon) is accepted identically to `cat:food`.
  Min/max range is validated upfront — `min > max` raises a clear error instead of silently returning an empty list.
  Internally, `buildPredicate()` uses `Predicate.and()` chaining so each active filter adds an AND condition,
  keeping all filter logic in a single source of truth shared across commands.
  44 unit tests cover all filter paths and edge cases.

- **SortCommand & TransactionSorter** (PR [#27](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/27)): Global sort state (`sort amount desc`) persists across all subsequent
  `list` and `summarize` calls until changed. Supports `amount`/`date` fields, `asc`/`desc` directions,
  `reset` (clears sort), and `show` (displays current order). `TransactionSorter` is a stateless utility;
  global state lives in `TransactionManager`, keeping sort logic decoupled from display logic.

- **Storage Management** (PR [#57](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/57)): `export`, `import`, and
  `clear` commands with `CsvStorageManager`. Export writes all transactions to CSV; import supports merge or replace
  mode; clear prompts for confirmation (or `--force`). CSV parsing handles quoted fields, escaped quotes, commas in
  descriptions, and newlines. Import validates each row independently — malformed rows are skipped with error messages
  while valid rows are imported.

- **Autosave & Crash Recovery** (PR [#64](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/64)): Implemented
  `AutoSaveManager` to persist transactions to `data/rlad.csv` after every mutating command (add, delete, modify,
  clear, import) and restore them automatically on startup. Originally used pipe-delimited `rlad.txt`; later migrated
  to CSV in PR [#139](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/139) to fix silent data loss when pipe
  characters appeared in category or description fields. 5 unit tests cover round-trip, empty file, null category,
  pipe-in-description, and missing-file scenarios.

- **Budget Persistence** (PR [#139](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/139)): Budget entries now
  survive app restarts via `data/budgets.csv`. Reuses `CsvStorageManager.exportToCsv()` for saving; load
  preserves original hash IDs. Existing `rlad.txt` autosave files are automatically migrated to `rlad.csv` on first
  launch.

- **HelpCommand** (PR [#70](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/70)): Wired up `HelpCommand` to
  dispatch to per-command manual methods in `Ui`. Added missing manual methods for sort, budget, filter, search, and
  help. Fixed incorrect syntax hint (`$action help` → `help <command>`). 20 unit tests cover all supported topics and
  unknown-command error handling.

- **Error Handling & Input Validation** (PR [#63](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/63)): Replaced
  generic "Invalid arguments" messages with specific, usage-hinting error messages for sort, list, filter, export, and
  import. Added strict calendar-date validation to reject impossible dates (e.g., Feb 30) that would previously be
  silently rounded. Fixed null pointer exceptions when filtering transactions with null categories.

- **Amount Validation in AddCommand** (PR [#69](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/69)): Reject zero
  and negative amounts (previously only `ModifyCommand` validated this). Cap maximum amount at 10,000,000 to prevent
  precision and display issues.

- **Assertions & Logging** (PR [#39](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/39)): Added `assert`
  statements and `java.util.logging` to `SortCommand` and `TransactionSorter` for defensive programming.

- **Enabled Runtime Assertions** (PR [#65](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/65)): Added `-ea` JVM
  flag to the Gradle `run` task so assertions are active during normal app execution (previously active only during
  tests).

### Bug fixes

- **Null category NPE** (PR [#74](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/74)): Fixed null pointer
  exceptions in `SearchCommand` and `ModifyCommand` when a transaction has no category. Also fixed `ExportCommand`
  crash when `--file ""` (empty quoted argument) was passed, and unified strict date parsing in `AddCommand` and
  `ModifyCommand`.

- **Export/Import positional args** (PR [#134](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/134)): `export
  [filename]` and `import <filename> [merge]` now parse positional arguments instead of requiring `--file`/`--merge`
  flags, matching the documented UG syntax. Updated tests accordingly.

- **Search positional arg** (PR [#136](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/136)): `search <keyword>`
  now works as documented — removed the undocumented `--keyword` flag requirement from `SearchCommand`. Updated tests
  to use positional syntax.

- **List space-after-colon parsing** (PR [#137](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/137)): `list min:
  50 max: 200` (space after colon) now parses correctly — the parser normalises `key: value` to `key:value` before
  splitting. Fixes #125.

- **ModifyCommand quoted description** (PR [#170](https://github.com/AY2526S2-CS2113-W13-4/tp/pull/170)): Fixed
  `ModifyCommand` silently dropping quoted description values containing spaces (e.g.,
  `description="grocery run"`).
