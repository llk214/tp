# Roshan Premil - Project Portfolio Page

### Project: RLAD (Record Losses And Debt)

RLAD is a CLI-based personal finance tracker for users who prefer typing over GUI navigation. Users can add, delete, modify, search, filter, sort, and summarize transactions from the terminal. It is written in Java and has about 6 kLoC.

Given below are my contributions to the project.

* **New Feature**: Added the `delete` command to remove transactions by HashID.
  * What it does: Deletes a transaction identified by its unique 6-character HashID.
  * Justification: Users need a way to remove incorrect or unwanted transactions.
  * Highlights: Validates that exactly one argument is provided, converts HashID to lowercase for case-insensitive matching, and removes from both `ArrayList` and `HashMap` in `TransactionManager` atomically.

* **New Feature**: Added the `modify` command to update existing transactions in-place.
  * What it does: Updates one or more fields (`type`, `amount`, `date`, `category`, `description`) of a transaction using `field=value` syntax, e.g., `modify a7b2c3 amount=25.00 description="Lunch with friends"`.
  * Justification: Without this, users would have to delete and re-add a transaction to correct a single field, losing the original HashID.
  * Highlights: Implements a quote-aware tokenizer to handle multi-word values in double quotes. Validates field names against a whitelist, rejects unknown fields, and preserves the original HashID via `setHashId()`. Required careful design to update both `ArrayList` and `HashMap` atomically.

* **New Feature**: Added the `summarize` command for financial overviews.
  * What it does: Calculates total credit, total debit, and net balance, grouped by category. Supports the same filter flags as `list` (type, category, amount, date range).
  * Justification: Users need at-a-glance financial summaries without manually adding up transactions.
  * Highlights: Uses `BigDecimal` for all aggregation to avoid floating-point display errors. Reuses `FilterCommand.buildPredicate()` to avoid duplicating filter logic.

* **New Feature**: Added the `search` command to find transactions by keyword.
  * What it does: Searches across description, category, HashID, and formatted amount fields using case-insensitive substring matching.
  * Justification: With many transactions, users need a quick way to locate specific entries without scrolling through the full list.

* **Enhancement**: Extracted `AmountValidator` shared utility class.
  * Consolidated duplicated amount validation logic from `AddCommand` and `ModifyCommand` into a single utility. Rejects scientific notation, amounts with >2 decimal places, and amounts that round to $0.00.

* **Bug Fixes (PE-D)**: Fixed 15 bugs reported during PE dry run across multiple components:
  * Input validation: reject extra arguments in `delete`/`sort`, validate min>max before filtering, reject NaN/Infinity in storage loading, reject zero-amount CSV imports, validate type filter values
  * Case sensitivity: accept case-insensitive type values in `modify` and `filter`
  * Edge cases: fix infinite loop on EOF (piped input), fix negative budget display, warn on budget overwrite, handle empty filter keys, strip legacy `--file` flag in import/export
